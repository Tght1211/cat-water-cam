# 猫咪饮水摄像头

纯本地的猫咪饮水监控：普通 USB 摄像头 + Mac Mini，认猫在流动水碗停留即录一段视频，记录每日喝水次数。**网页平台（可局域网访问）** 看实时画面 / 喝水趋势 / 视频回放并对每段点 👍/👎 攒训练数据，攒够后**一键训练分类器**自我迭代；**猫来喝水自动发邮件**（带照片 + 今日次数 + 周/月趋势）。画面只在本机与同局域网内流转。

## 功能特性

- 🐱 **认猫 + 行为判定**：YOLO 认猫，加一个无需训练的「简单模型」（画面变化 + 灰蓝猫色块）做兜底，宁可多录候选。猫在水碗 ROI 停留够久即触发。
- 🎥 **整段会话录制**：从猫开始喝录到离开（变长），开头还补几秒「凑近过程」做 pre-roll；编码 **H.264**（浏览器可直接播放），只留最近若干段。
- 📺 **流畅实时画面**：MJPEG 视频流（非每秒刷快照）；采集 / 检测 / 录制分线程，预览不被推理拖慢。
- 🖥️ **现代化网页**：分标签页仪表盘（总览 / 趋势 / 视频 / 训练）；视频列表**懒加载**——先看封面缩略图，点了才播，不再全部一起转圈。
- 📊 **喝水趋势**：网页端原生 SVG 图表（近 7 天 / 近 30 天可切换），今日次数、各次时间点、日均。
- 🏷️ **标注即数据**：每段视频点 👍真喝水 / 👎没喝；标「没喝」会从喝水计数里自动剔除。标注帧落盘成训练集。
- 🧠 **一键训练 / 模型版本 / 自我迭代**：网页用标注帧训练「真喝水 / 没喝」分类器（YOLOv8-cls）。训练页实时显示**待标注 / 已标注未训练 / 已标注已训练**，无新标注就不重复练；每次训练产出带准确率的**版本 vN**，可随时**选哪个生效**（或停用只用简单模型）。生效模型会在录制前确认「真喝水」，过滤好奇凑近的误触。
- 📧 **喝水邮件提醒**：触发即发 HTML 邮件（触发照片 + 今日时间点 + 周/月趋势图 + 平台链接），带最小间隔限流防误判刷屏。
- 🌙 **夜间/弱光**：无红外也尽量可用——判暗后做弱光增强（CLAHE），夜间靠「画面变化」识别行为；也可设为天黑不记录。

## 安装

```bash
python3 -m venv .venv
.venv/bin/pip install -e .
.venv/bin/pip install -r requirements.txt
```

首次运行会自动下载一次 YOLO 权重（`yolov8n.pt`）；一键训练首次会再下载分类基座（`yolov8n-cls.pt`）。

## 运行

```bash
# 真实采集（需摄像头）
.venv/bin/python -m catcam

# 演示/试标注（无摄像头，自动塞示例视频，只起网页）
.venv/bin/python -m catcam.demo
```

- 本机访问 `http://127.0.0.1:8000`
- 同局域网设备访问 `http://<本机IP>:8000`（启动日志会打印实际地址）

## 配置

首次运行生成 `config.json`（已 gitignore，含凭据，不进库）。常用项：

| 键 | 说明 |
| --- | --- |
| `camera_index` / `video_source` | 摄像头编号；或填一段视频文件路径用录像代替摄像头测试 |
| `bowl_roi` | 水碗区域，0–1 比例 `[x1,y1,x2,y2]`，与分辨率无关；对不准就按画面比例改 |
| `min_overlap_ratio` | 猫框 ∩ 水碗 / 水碗面积 ≥ 此值才算「在水碗」 |
| `dwell_seconds` / `cooldown_seconds` | 停留判定秒数 / 两次事件最小间隔 |
| `max_clips` | 最多保留段数 |
| `record_session` | 整段会话录制（变长，默认 `true`）；`false` 则用固定 `clip_seconds` |
| `preroll_seconds` / `session_end_grace_seconds` / `max_session_seconds` | 触发前补录 / 离开多久收尾 / 单段封顶 |
| `web_host` / `web_port` | `0.0.0.0` 暴露给局域网；`127.0.0.1` 只本机 |
| `mail_enabled` / `smtp_*` / `mail_to` | 邮件开关、发件 SMTP（密码填邮箱**授权码**）、收件地址 |
| `mail_min_interval_seconds` | 邮件最小间隔（限流） |
| `night_brightness_threshold` / `record_at_night` | 判暗阈值 / 天黑是否仍记录 |
| `detect_interval_seconds` | 检测节奏（与采集/预览解耦） |

### 邮件提醒

发件用支持 SMTP 的邮箱（如 QQ 邮箱，`smtp.qq.com:465` SSL），密码填邮箱**授权码**（不是登录密码）。示例：

```json
"mail_enabled": true,
"smtp_host": "smtp.qq.com",
"smtp_port": 465,
"smtp_user": "你的发件@qq.com",
"smtp_password": "邮箱授权码",
"mail_to": "你的收件@example.com"
```

### 模型训练 / 自我迭代

1. 网页上对每段视频点 👍真喝水 / 👎没喝（标注帧自动落盘到 `data/training/{drinking,not_drinking}/`）。
2. 「训练」页点 **开始训练** → 用标注帧训练分类器，登记成版本 `vN`（`data/models/registry.json`），显示验证集准确率。
3. **训练不自动生效**：在「模型版本」点「设为生效」选用某版本，默认进**测试模式（shadow）**——只预测、**不拦截录制**，简单模型继续兜底全录；在「视频」里看模型对每段判得准不准，攒「实战命中率」。
4. 等它够准了再切**过滤模式（gate）**，让它过滤「好奇凑近没喝」的误触；不准就别切，免得漏录真喝水。
5. 标得越多、重训越多越准；没有新标注时不会重复训练。

### 夜间说明

此摄像头无红外，全黑环境弱光增强也只能拉出轮廓、靠运动识别，可靠性有限；水碗旁放个小夜灯会显著改善。

## 已知范围

「好奇凑近没喝」会被误记——这是**有意为之**：先用简单模型多录候选，靠标注 + 一键训练逐步修正。隐私时段自动关闭、网页画框选区域、开机自启为后续阶段。
