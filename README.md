# 猫咪饮水摄像头

纯本地的猫咪饮水监控：普通 USB 摄像头 + Mac Mini，认猫在流动水碗停留即录一段视频，记录每日喝水次数。**网页平台（可局域网访问）** 看实时画面 / 喝水趋势 / 视频回放并对每段点 👍/👎 攒训练数据，攒够后**一键训练分类器**自我迭代；**猫来喝水自动发邮件**（带照片 + 今日次数 + 周/月趋势）。画面只在本机与同局域网内流转。

## 功能特性

- 🐱 **认猫 + 行为判定**：YOLO 认猫，加一个无需训练的「简单模型」（画面变化 + 灰蓝猫色块）做兜底，宁可多录候选。猫在水碗 ROI 停留够久即触发。
- 🎥 **整段会话录制**：从猫开始喝录到离开（变长），开头还补几秒「凑近过程」做 pre-roll；编码 **H.264**（浏览器可直接播放），只留最近若干段。
- 📺 **流畅实时画面**：MJPEG 视频流（非每秒刷快照）；采集 / 检测 / 录制分线程，预览不被推理拖慢。
- 🖥️ **现代化网页**：分标签页仪表盘（总览 / 趋势 / 视频 / 训练）；视频列表**懒加载**——先看封面缩略图，点了才播，不再全部一起转圈。
- 📊 **喝水趋势**：网页端原生 SVG 图表（近 7 天 / 近 30 天可切换），今日次数、各次时间点、日均。
- 🏷️ **标注即数据**：每段视频点 👍真喝水 / 👎没喝；标注帧落盘成训练集。**今日喝水次数 / 趋势只数被确认「喝水」的段**（未标注 / 标「没喝」都不计）。
- 🧠 **一键训练 / 模型版本 / 自我迭代**：网页用标注帧训练「真喝水 / 没喝」分类器（YOLOv8-cls）。训练页实时显示**待标注 / 已标注未训练 / 已标注已训练** + **训练进度条与预估时间**，无新标注就不重复练；每次训练产出带准确率的**版本 vN**，可随时**选哪个生效**（或停用只用简单模型）。生效模型会在录制前确认「真喝水」，过滤好奇凑近的误触。训练时自动做**类别平衡**（喝/没喝样本拉到接近 1:1），避免「啥都判没喝」。
- 🤖 **AI 当裁判（默认开，需填 `ai_api_key`）**：接一个外部视觉大模型（OpenRouter / 任意 OpenAI 兼容接口），**录一段后由它判「猫是否真喝水」——这是「发邮件 + 记次数」的唯一权威**：判喝水才发邮件、才计入；其余只留作素材。判定同时写进训练集（省掉人工标一大堆无用片段）。失败 fail-open 不影响录制；人工标注优先、不被覆盖。⚠️ 填了 key 开始判定后画面帧会上传所配服务，详见下文。规划中第二阶段用本地小视频模型（VideoMAE）接管，画面回归全本地。
- 📧 **喝水邮件提醒**：**AI 判「真喝水」后**发 HTML 邮件（触发照片 + 今日时间点 + 周/月趋势图 + 平台链接），带最小间隔限流防误判刷屏。
- 🌙 **夜间/弱光**：无红外也尽量可用——判暗后做弱光增强（CLAHE），夜间靠「画面变化」识别行为；也可设为天黑不记录。

## 安装

```bash
python3 -m venv .venv
.venv/bin/pip install -e .
.venv/bin/pip install -r requirements.txt
```

首次运行会自动下载一次 YOLO 权重（`yolov8n.pt`）；一键训练首次会再下载分类基座（`yolov8n-cls.pt`）。

## 运行

```bash
# 真实采集（需摄像头）
.venv/bin/python -m catcam

# 演示/试标注（无摄像头，自动塞示例视频，只起网页）
.venv/bin/python -m catcam.demo
```

- 本机访问 `http://127.0.0.1:8000`
- 同局域网设备访问 `http://<本机IP>:8000`（启动日志会打印实际地址）

## 配置

首次运行生成 `config.json`（已 gitignore，含凭据，不进库）。常用项：

| 键 | 说明 |
| --- | --- |
| `camera_index` / `video_source` | 摄像头编号；或填一段视频文件路径用录像代替摄像头测试 |
| `bowl_roi` | 水碗区域，0–1 比例 `[x1,y1,x2,y2]`，与分辨率无关；对不准就按画面比例改 |
| `min_overlap_ratio` | 猫框 ∩ 水碗 / 水碗面积 ≥ 此值才算「在水碗」 |
| `dwell_seconds` / `cooldown_seconds` | 停留判定秒数 / 两次事件最小间隔 |
| `max_clips` | 最多保留段数 |
| `record_session` | 整段会话录制（变长，默认 `true`）；`false` 则用固定 `clip_seconds` |
| `preroll_seconds` / `session_end_grace_seconds` / `max_session_seconds` | 触发前补录 / 离开多久收尾 / 单段封顶 |
| `web_host` / `web_port` | `0.0.0.0` 暴露给局域网；`127.0.0.1` 只本机 |
| `mail_enabled` / `smtp_*` / `mail_to` | 邮件开关、发件 SMTP（密码填邮箱**授权码**）、收件地址 |
| `mail_min_interval_seconds` | 邮件最小间隔（限流） |
| `night_brightness_threshold` / `record_at_night` | 判暗阈值 / 天黑是否仍记录 |
| `detect_interval_seconds` | 检测节奏（与采集/预览解耦） |
| `ai_label_enabled` | AI 裁判/标注开关（默认 `true`，但需填 `ai_api_key` 才生效；判定时画面帧会上传所配服务）。无 key = 不发邮件、次数恒 0 |
| `ai_base_url` / `ai_api_key` / `ai_model` | 外部模型接口地址 / 密钥 / 模型名（OpenRouter 或任意 OpenAI 兼容服务/本地代理） |
| `ai_label_frames` | 每段抽几帧送给模型判断（默认 3） |

### 邮件提醒

发件用支持 SMTP 的邮箱（如 QQ 邮箱，`smtp.qq.com:465` SSL），密码填邮箱**授权码**（不是登录密码）。示例：

```json
"mail_enabled": true,
"smtp_host": "smtp.qq.com",
"smtp_port": 465,
"smtp_user": "你的发件@qq.com",
"smtp_password": "邮箱授权码",
"mail_to": "你的收件@example.com"
```

### 从 0 到 1 训练属于自己的「小猫喝水识别模型」

刚装好时还没有模型，靠无需训练的「简单模型」兜底**多录候选**（宁可多录、好奇凑近也录下来）。
目标是用这些录像攒出标注数据，训出一个**专属你家猫 + 你家水碗**的分类器，逐步替简单模型把关。

整条「自我迭代」闭环：

1. **攒候选**：开着 `python -m catcam`，猫每次凑近水碗就会录一段（变长会话录制）。
2. **打标签**（攒训练数据）—— 两条路，可混用：
   - **人工标**：在「视频」页对每段点 👍真喝水 / 👎没喝。
   - **AI 辅助标**（可选）：开 `ai_label_enabled`，录一段就自动让外部视觉大模型判「喝/没喝」直接进训练集，省掉人工标一大堆无用片段（见下「AI 辅助标注」）。
   - 两种方式标注帧都会落盘到 `data/training/{drinking,not_drinking}/`；**人工标注优先**，已人工标的段 AI 不覆盖、你也能随时手动翻转。
3. **训练**：「训练」页点 **开始训练** → 用标注帧训练分类器（YOLOv8-cls），带**进度条 + 预估时间**；训练自动做**类别平衡**（喝/没喝拉到接近 1:1，避免失衡把模型带歪成「啥都判没喝」）。训完登记成版本 `vN`（`data/models/registry.json`），显示验证集准确率。
4. **生效（不自动）**：在「模型版本」点「设为生效」选某版本，默认进**测试模式（shadow）**——只预测、**不拦截录制**，简单模型继续兜底全录；在「视频」里看模型对每段判得准不准，攒「实战命中率」。
5. **够准了再过滤**：切**过滤模式（gate）**让模型过滤「好奇凑近没喝」的误触；不准就别切，免得漏录真喝水。
6. 标得越多、重训越多越准；没有新标注时不会重复训练。

> 判定口径（也是 AI 提示词里写死的）：这是台**自动饮水机，水面一直有波纹**，所以**水波不能当喝水证据**；真正算喝水的是**猫凑近 + 伸出粉红舌头 + 舔水动作**。

### AI 当裁判 / 辅助标注（接 OpenRouter / 任意 OpenAI 兼容模型）

接一个**外部视觉大模型**判每段「猫是否真喝水」：**它是「发邮件 + 记次数」的唯一权威**（判喝水才发、才计入，其余只留作素材），同时把判定写进训练集、省掉你手动标几百段无用录像。
**默认开，但需填 `ai_api_key` 才生效**（没填 = 无裁判，不发邮件、次数恒 0）。⚠️ 判定时会把猫的画面帧上传到你配置的服务，与「画面只在本机/局域网」相悖，请自行权衡；规划中第二阶段用本地小视频模型（VideoMAE）接管后即可关停 VLM 回归全本地（见 `docs/superpowers/specs/2026-06-26-video-action-judge-design.md`）。

在 `config.json` 里配：

```json
"ai_label_enabled": true,
"ai_base_url": "https://openrouter.ai/api/v1",
"ai_api_key": "你的-key",
"ai_model": "google/gemma-4-31b-it:free",
"ai_fallback_models": [
  "nvidia/nemotron-nano-12b-v2-vl:free",
  "google/gemma-4-26b-a4b-it:free",
  "nvidia/nemotron-3-nano-omni-30b-a3b-reasoning:free"
],
"ai_label_frames": 3
```

- **接口**：走 OpenAI 兼容协议，`ai_base_url` 可指向 OpenRouter、其它兼容服务、或本地代理（如 gpt-load）。需选**支持视觉输入**的模型。
- **多模型轮换 + 兜底**：`ai_model` 是主模型，`ai_fallback_models` 列额外模型，一起组成模型池。**每次调用轮换起始模型**（分摊各免费模型的每日额度，少撞 429），**某个失败就顺位换下一个**，全池都失败才算这段标注失败。免费模型限流严重时，多配几个能明显提升成功率。
- **怎么工作**：录完一段 → 后台抽 `ai_label_frames` 帧 → 问模型「猫是否真喝水」→ 拿到 `{喝/没喝, 置信度, 理由}` 直接写标注并进训练集。视频页会显示来源徽标 🤖 AI（含置信度/理由）/ ✋ 人工。
- **稳**：调用失败（限流/网络/解析失败）只记日志、该段保持未标注，**绝不影响录制**。免费模型常有每日额度/限流（429），到时那段就先空着，可重录或手动标，或换自己的付费 key / 别的模型。
- **隐私/网络**：若机器走了 VPN（SOCKS 代理），程序已设 `trust_env=False` 直连，不被系统代理影响。

### 夜间说明

此摄像头无红外，全黑环境弱光增强也只能拉出轮廓、靠运动识别，可靠性有限；水碗旁放个小夜灯会显著改善。

## 已知范围

「好奇凑近没喝」会被误记——这是**有意为之**：先用简单模型多录候选，靠标注 + 一键训练逐步修正。隐私时段自动关闭、网页画框选区域、开机自启为后续阶段。
